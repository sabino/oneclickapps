Supabase for CapRover

This repository publishes the full self-hosted Supabase stack as a CapRover one-click app. The generated supabase.yml is the final standalone artifact that CapRover consumes.

The source is generated because Supabase is a large multi-service deployment that needs custom config files, custom image build steps, and a few CapRover-specific adjustments to stay maintainable.

What this app deploys

Profile Full Supabase stack
Public service Kong gateway
Generated artifact Standalone supabase.yml

How the generator works

  • The generator uses the official Supabase self-hosted stack as the behavior reference.
  • Custom assets and Dockerfile templates are kept in this repository so the final YAML stays self-contained.
  • The generator emits one standalone CapRover template with pinned component versions exposed as variables.
  • A small optional Studio patch adds support for custom OpenAI-compatible base URLs, which is useful for Azure OpenAI.

Operator notes

  1. After deployment, enable HTTPS on the public supabase service in CapRover.
  2. If your server still uses the default Docker Swarm overlay /24, very large one-click stacks can exhaust available VIPs.
  3. If that happens, either remove unused services or migrate the shared overlay network capacity before retrying.
